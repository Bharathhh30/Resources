
**Relational Databases (SQL)**

- **CRUD Operations**: Create, Read, Update, and Delete data in relational databases.
    
- **Joins**: Combine data from multiple tables using INNER, OUTER, LEFT, RIGHT, and CROSS joins.
    
- **Aggregations**: Perform calculations like SUM, AVG, COUNT, MIN, and MAX for data analysis.
    
- **Indexing**: Improve query performance with proper indexing techniques.
    
- **Transactions**: Ensure data consistency with ACID properties (COMMIT, ROLLBACK, SAVEPOINT).
    
- **Views**: Create and manage virtual tables for simplified, reusable queries.
    
- **Stored Procedures**: Develop reusable SQL scripts for complex logic and operations.
    

**Non-Relational Databases (NoSQL)**

- **Overview**: Differences between relational and non-relational databases.
    
- **Types of NoSQL Databases**: Document-based, key-value, column-family, and graph databases.
    
- **Document Databases**: Working with MongoDB for JSON-like data storage and retrieval.
    
- **Key-Value Stores**: Using Redis for caching and real-time applications.
    
- **Column-Family Databases**: Leveraging Cassandra for large-scale, distributed data.
    
- **Graph Databases**: Modeling relationships with Neo4j.
    
- **Scalability & Performance**: Optimizing NoSQL databases for high availability and performance.
    
- **Use Cases**: Identifying scenarios where NoSQL is more effective than SQL.
